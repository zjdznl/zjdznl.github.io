查理·芒格说他靠大约 100 个思维模型做决策。他没列过清单，但他的追随者们整理了出来。

我不打算列 100 个。你也不会看。

我从里面挑了 5 个。标准不是「最著名的」，是「程序员日常就能用的」。

你会发现一个规律：这些模型都不是用来「知道」的，是用来「出手」的。它们不是知识，是扳机——在关键时候提醒你换个角度看问题。

第一个：乘法为零效应——你的系统里有没有「归零」bug？
#

数学上，任何数乘以零等于零。翻译成人话：一个系统里如果某个关键环节完全失效，其他部分再强也没用。

你写的代码里有没有这种 bug？你设计的架构里有没有这种单点？你的职业生涯里有没有这种致命弱点？

很多程序员花大量时间优化非关键路径——把某个接口的响应时间从 200ms 降到 50ms，把代码写得无比优雅——但数据库没有备份。优化 99 个环节，最后一个环节是零，整体结果还是零。

「会让我归零的东西，优先级永远高于能让我变好的东西。」

下次排需求优先级的时候，不要只按「做了能提升多少」排。也按「不做会死多惨」排。前者让你更好，后者让你活着。

第二个：反向思考——别只想怎么成功，先想怎么失败
#

芒格最爱引用的原则：「倒过来想，总是倒过来想。」

做技术方案的时候，你的第一反应是什么？「怎么把这个需求做出来」。正常的。但芒格会问你：「怎么做这个方案一定会失败？」

把所有可能搞砸的点列出来——依赖方挂了怎么办？数据不一致怎么办？流量突然飙十倍怎么办？——然后一个一个堵上。

这跟写代码的 defensive programming 是一个道理。 你不是在预测未来，你是在给自己留后路。好代码不是「能跑」，是「跑不坏」。

反向思考最狠的地方在于：它绕过了你的确认偏误。你天然倾向于找支持自己方案的理由。但「怎么一定会失败」这个问题，逼你站在自己的对立面。

第三个：安全边际——别把系统推到极限
#

桥梁设计的承重是预计最大荷载的三到五倍。芒格买股票也是这个思路——只在价格远低于内在价值时才出手。两者底层逻辑一样：现实会比你预估的更糟。

你的服务能撑多少 QPS？你说 10000。实际测过吗？测过。但真实流量分布跟压测不一样——有突发尖峰、有慢请求、有上游重试风暴。你说撑 10000，可能 8000 就挂了。

这不是悲观主义，是工程常识。给自己留至少 30% 的 buffer。不只是系统容量，汇报工期也一样——你说两周，但突发事件、依赖方延期、需求变更，随便来一个就超期。

任何一个没有留余量的承诺，都是在赌一切都按照你的预期发生。而一切永远不会按照你的预期发生。

第四个：能力圈——知道「这不该我搞」比知道「怎么搞」更值钱
#

巴菲特和芒格一辈子不投高科技公司，因为他们承认自己不懂。互联网泡沫的时候别人都在嘲笑他们落伍，泡沫破了之后别人都在后悔没学他们。

程序员这个圈子里，承认「我不懂」是一件非常难的事。因为你的工作就是「搞定问题」，你习惯了什么都能学、什么都能搞。但「什么都能搞」和「什么都该我搞」是两码事。

你的能力圈是什么？不是你「学过」的东西，是你「能独立做决策」的东西。

学过两天 Rust 不叫 Rust 在你的能力圈里。能用 Rust 独立上线一个生产系统、能判断一个 Rust 项目的架构对不对——这才叫进入了能力圈。

能力圈外面的事，只有两种处理方式：要么不进，要么先学到能用再进。 最危险的是第三种——没进强进，凭感觉拍板。

你不需要成为什么都会的人。你需要知道你会什么、不会什么，并且诚实地按这个边界做事。

第五个：贝叶斯更新——把「我错了」从灾难变成流程
#

贝叶斯定理的公式你不用记。记住一个逻辑就行：先有一个初始判断，拿到新信息后，更新判断。

程序员对「更新」这个词有天然的亲切感。你的代码每天在更新，你的认知也应该每天在更新。

但大多数人做不到。因为更新判断意味着承认之前的判断不准。承认自己错了这件事，在心理上比改 bug 难一百倍。

换个角度看就不一样了。把「更新判断」当成一个技术动作，而不是道德审判。 你改代码的时候不会觉得「我之前写的代码真丢人」，你知道代码就是需要不断改的。判断也是一样——不是因为之前的你蠢，是因为现在的你有更多信息。

你今年年初对 AI programming 的判断，跟现在对 AI programming 的判断一模一样——这不是一致性，这是僵化。这不是立场坚定，这是没有学习。

好的程序员不是一次写对代码的人，是发现错了马上改的人。好的决策者同理。

这些模型怎么用
#

不是背下来。是遇到事的时候，脑子里自动弹一个扳机：

  • 做方案的时候，弹「反向思考」：怎么做会失败？
  • 评估风险的时候，弹「乘法为零」：有什么环节能让我归零？
  • 给承诺的时候，弹「安全边际」：留够 buffer 了吗？
  • 被要求做决定的时候，弹「能力圈」：这在我的圈内吗？
  • 发现判断错了的时候，弹「贝叶斯更新」：新的输入来了，输出该变了。

思维模型不是知识，是习惯。 第一次用会觉得刻意，第一百次就成了本能。

芒格说过一句话：「拿着一把锤子的人，看什么都像钉子。」他要你做的不是扔掉锤子，是在工具箱里多放几把工具。

下一次你做技术决策的时候，试试从这五个里抽一个出来用。用一次，你就知道它比任何「最佳实践」都好使。
